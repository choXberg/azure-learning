## High Availability & Scalability (Cloud Benefits)

### High Availability

* Ensures services are **always accessible** (minimal downtime)
* Azure provides **uptime guarantees via SLAs**
* Designed to handle:

  * failures
  * outages
  * unexpected events

👉 Key idea:

* System stays available even if something breaks

👉 Exam tip:

* High availability = **SLA (Service Level Agreement)**

---

### Scalability

* Ability to **adjust resources based on demand**
* Scale up when demand increases
* Scale down when demand decreases

👉 Benefit:

* Handle peak loads
* Avoid overpaying (cost efficiency)

---

### Types of Scaling

#### Vertical Scaling (Scale Up/Down)

* Increase/decrease **power of a single resource**

  * more CPU
  * more RAM

👉 Example:

* Upgrade a VM to a bigger size

---

#### Horizontal Scaling (Scale Out/In)

* Increase/decrease **number of resources**

  * more VMs
  * more containers

👉 Example:

* Add more VM instances during high traffic

---

## 🧠 Quick Memory Guide

* High Availability → **uptime + SLA**
* Scalability → **adapt to demand**
* Vertical → **bigger machine**
* Horizontal → **more machines**

---

## 🎯 Exam Triggers

👉 "Guaranteed uptime"
→ High Availability (SLA)

👉 "Add more VMs to handle traffic"
→ Horizontal scaling

👉 "Increase CPU/RAM of a VM"
→ Vertical scaling

👉 "Reduce cost when demand drops"
→ Scalability (scale down)
