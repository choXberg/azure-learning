## Security & Governance (Cloud Benefits)

### Security

* Cloud provides **built-in security protections**
* Protection against threats like:

  * DDoS attacks
* Security responsibility depends on service model:

  * IaaS → you manage OS, patches, apps
  * PaaS/SaaS → provider manages more security

👉 Key idea:

* More control = more responsibility (IaaS)
* Less control = provider handles more (PaaS/SaaS)

---

### Governance

* Ensures resources meet:

  * company standards
  * regulatory requirements

👉 Supported by:

* templates (standardized deployments)
* policies
* compliance auditing

👉 Key idea:

* Keep environment **consistent and compliant**

---

### Compliance & Auditing

* Continuous monitoring of resources
* Detects:

  * misconfigurations
  * policy violations

👉 Benefit:

* Issues can be identified and fixed quickly

---

### Automation

* Automatic:

  * updates
  * patches
  * policy enforcement

👉 Benefit:

* Improves both **security and governance**

---

## 🧠 Quick Memory Guide

* Security → **protect systems & data**
* Governance → **control & standards**
* IaaS → more responsibility
* PaaS/SaaS → provider handles more

---

## 🎯 Exam Triggers

👉 "Who is responsible for OS patches?"
→ Depends on service model (IaaS vs PaaS/SaaS)

👉 "Ensure compliance with standards"
→ Governance

👉 "Detect non-compliant resources"
→ Auditing

👉 "Protection against attacks like DDoS"
→ Cloud security
