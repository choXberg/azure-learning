## Platform as a Service (PaaS)

### Core Idea

* Middle ground between **IaaS and SaaS**
* Provides a **complete development platform**

👉 You focus on **building applications**, not managing infrastructure

---

### Responsibility Split

👉 Cloud provider manages:

* hardware & datacenter
* operating system
* middleware
* runtime environment
* development tools

👉 You manage:

* application code
* data
* access & configuration

👉 Key idea:

* **Less control than IaaS, less effort required**

---

### Benefits

* No need to manage OS or infrastructure
* Faster development
* Built-in features:

  * scalability
  * high availability
  * multitenancy

---

### Typical Use Cases

#### Development Framework

* Build and deploy applications quickly
* Use built-in components and services

#### Analytics / Business Intelligence

* Analyze data
* Find patterns and insights
* Support decision-making

---

## 🧠 Quick Memory Guide

* PaaS → **focus on code**
* Provider manages → OS + runtime
* You manage → app + data

---

## 🎯 Exam Triggers

👉 "No need to manage OS or patches"
→ PaaS

👉 "Focus on application development"
→ PaaS

👉 "Which model provides a platform for developers?"
→ PaaS

👉 "Managed database / runtime environment"
→ PaaS
