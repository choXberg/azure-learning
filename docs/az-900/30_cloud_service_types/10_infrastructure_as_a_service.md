## Infrastructure as a Service (IaaS)

### Core Idea

* Most flexible cloud service model
* Provides **maximum control** over resources

👉 You are essentially **renting hardware in the cloud**

---

### Responsibility Split

👉 Cloud provider manages:

* physical hardware
* datacenter
* networking (infrastructure level)
* physical security

👉 You manage:

* operating system (install, configure, patch)
* applications
* data
* network configuration (logical)
* databases & storage setup

👉 Key idea:

* **More control = more responsibility**

---

### Benefits

* Full control over environment
* High flexibility
* Suitable for custom configurations

---

### Typical Use Cases

#### Lift-and-Shift Migration

* Move existing on-prem systems to cloud
* Minimal changes required

#### Development & Testing

* Quickly create and remove environments
* Full control over setup

---

## 🧠 Quick Memory Guide

* IaaS → **maximum control**
* You manage → OS, apps, data
* Provider manages → hardware

---

## 🎯 Exam Triggers

👉 "You need full control over OS"
→ IaaS

👉 "Migrate existing system without changes"
→ Lift-and-shift (IaaS)

👉 "Who manages the OS?"
→ Customer (IaaS)

👉 "Who manages physical hardware?"
→ Cloud provider
