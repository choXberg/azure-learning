# Azure Service Health

## What is Azure Service Health?

**Azure Service Health** helps you monitor the health of Azure services and your specific resources.

It provides visibility from **global Azure status** down to **individual resources**, so you can quickly understand and respond to issues.

---

## Three Health Views

Azure Service Health combines three levels of insight:

### 🌍 Azure Status
- Provides a **global view** of Azure health
- Covers all services and regions
- Useful for detecting **widespread outages**

👉 Use when:  
You hear about a major Azure issue and want to check if it’s real.

---

### 🧭 Service Health
- Focuses on **services and regions you use**
- Personalized based on your subscription
- Shows:
  - outages
  - planned maintenance
  - health advisories

- Supports **alerts and notifications**

👉 Use when:  
You want to know how Azure issues affect *your environment*.

---

### 🔍 Resource Health
- Provides status for **individual resources**
- Example: a specific VM, database, or service

Shows:
- If the resource is healthy or unavailable
- Whether the issue is:
  - caused by Azure
  - caused by your configuration

👉 Use when:  
You troubleshoot a **specific resource problem**.

---

## Combined Value

Together, these three views provide:

- End-to-end visibility (global → service → resource)
- Faster troubleshooting
- Better understanding of outages and impact

---

## Additional Features

- **Historical data** → identify recurring issues or trends
- **Alerts** → get notified automatically about relevant events
- **Support integration** → quick access to Azure support when incidents occur

---

## Operational Value

Azure Service Health helps you:

- Detect and respond to incidents faster
- Understand whether issues are **Azure-related or local**
- Minimize downtime and impact on workloads

---

## 🧠 Exam Tip

- Azure Status = **global (all Azure)**
- Service Health = **your services (personalized)**
- Resource Health = **specific resource**

👉 Think: **Global → Scoped → Individual**
