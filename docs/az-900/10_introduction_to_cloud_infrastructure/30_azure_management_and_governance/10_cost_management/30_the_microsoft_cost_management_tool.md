# 💰 Microsoft Cost Management (AZ-900)

## 🧠 Overview
- **Microsoft Cost Management** = tool to monitor, control, and optimize Azure costs
- Helps prevent unexpected charges
- Works across:
  - Subscriptions
  - Resource groups
  - Services

👉 Goal:
- Increase cost visibility
- Control spending
- Enable cost optimization

---

## 🔑 Core Features

### 1️⃣ Cost Analysis
- Visualize and analyze Azure costs

#### You can view costs by:
- Subscription
- Resource group
- Service
- Region
- Time (daily, monthly, yearly)

---

#### Use Cases:
- Identify cost trends
- Detect unexpected spikes
- Find idle/unnecessary resources
- Compare actual cost vs. budget

---

### 2️⃣ Cost Alerts

Central place for all cost-related alerts

#### Types of Alerts:

---

#### 📊 Budget Alerts
- Triggered when spending reaches a defined threshold
- Example:
  - Alert at 80% of budget

👉 Helps:
- Prevent overspending
- Take action early

---

#### 💳 Credit Alerts
- For Enterprise Agreements (EA)
- Triggered at:
  - 90% of credit usage
  - 100% of credit usage

---

#### 🏢 Department Spending Quota Alerts
- Also for EA customers
- Triggered when department reaches quota threshold
- Example:
  - 50% or 75% of quota

---

### 3️⃣ Budgets

- Define a **spending limit**
- Can be scoped to:
  - Subscription
  - Resource group
  - Service

---

#### Budget Capabilities:
- Set thresholds (e.g., 80%, 100%)
- Trigger alerts
- Send email notifications

---

#### Automation (Important!)
- Budgets can trigger actions:
  - Stop VMs
  - Scale resources
  - Shut down non-production environments

👉 Helps enforce cost control automatically

---

## 🧪 Practical Use Cases

- Detect cost increase after deployment
- Identify unused resources
- Validate tagging and cost allocation
- Track spending over time

---

## 🎯 Key Concepts (Exam Focus)

- Cost Management = **monitor + control Azure costs**
- Core components:
  - Cost Analysis
  - Alerts
  - Budgets
- Budgets can:
  - Trigger alerts
  - Trigger automation
- Alerts help prevent overspending

---
