## Azure Accounts & Subscriptions

### Core Idea

* To use Microsoft Azure, you need:

  * an **Azure account**
  * at least one **subscription**

👉 Key idea:

* Subscription = **billing + resource container**

---

### Azure Hierarchy

* **Account** → identity (user/company)
* **Subscription** → billing + logical container
* **Resource Group** → group of related resources
* **Resources** → actual services (VM, DB, etc.)

👉 Example:

* One account
  → multiple subscriptions (Dev, Test, Prod)
  → multiple resource groups
  → multiple resources

---

### Subscriptions

* Used to:

  * organize resources
  * manage costs
  * apply access control

👉 Common setup:

* Dev / Test / Production separation

---

### Azure Free Account

* 12 months free access to popular services
* Credit for the first 30 days
* Some services always free

👉 Important:

* Credit card required (verification only)
* No charges unless upgraded

---

### Azure Free Student Account

* Free services + credit (e.g. $100)
* No credit card required
* Includes developer tools

---

### Best Practice

* Monitor usage regularly
* Delete unused resources

👉 Key idea:

* Avoid unexpected costs

---

## 🧠 Quick Memory Guide

* Account → who you are
* Subscription → billing + container
* Resource Group → organize resources
* Resource → actual service

---

## 🎯 Exam Triggers

👉 "What is required to use Azure?"
→ Account + Subscription

👉 "What is used for billing?"
→ Subscription

👉 "Group related resources together"
→ Resource Group

👉 "Where are VMs, DBs created?"
