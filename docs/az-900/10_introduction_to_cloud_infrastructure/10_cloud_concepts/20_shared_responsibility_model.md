# 🔐 Shared Responsibility Model – Quick & Concise

## Definition:
Responsibility is shared between customer and Microsoft (Cloud Provider)

---
## ⚖️ Basic Principle
### On-Premises:
👉 You are responsible for everything
### Cloud:
👉 Responsibility is shared
---
## 🧠 Who does what?
### ☁️ Cloud Provider (always responsible)
- Physical data center
- Network (physical)
- Servers/Hosts
- Electricity, cooling, security
---
### 👤 Customer (always responsible)
- Data & information
- Access rights (Identity, Accounts)
- End devices (Laptop, Smartphone)
---
## 🔄 Depends on Service Model

👉 Important for exam!
| Model   | Description                 | Responsibility                        |
| -------- | --------------------------- | ------------------------------------ |
| **IaaS** | Infrastructure as a Service | Customer does much (OS, Apps, Updates) |
| **PaaS** | Platform as a Service       | Shared                              |
| **SaaS** | Software as a Service       | Provider does almost everything            |

![service-models](images/service-models.png)
---
## ⚡ Examples (very important)
### VM (IaaS):
-  👉 You patch OS + software
---
### Managed DB (PaaS):
- 👉 Provider manages database
- 👉 You are responsible for data
---
### 📌 Memory Aids
- 👉 "You own your data – always"
- 👉 The more managed (SaaS), the less responsibility
- 👉 IaaS = most responsibility with customer
---
### 🎯 Exam Focus
- Who is responsible for data? → ALWAYS customer
- Who is responsible for hardware? → ALWAYS provider
- Difference between IaaS / PaaS / SaaS
