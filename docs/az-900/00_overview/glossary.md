# AZ-900 Glossary

A concise collection of key terms and definitions for quick review.

---

## ☁️ Cloud Fundamentals

### Cloud Computing

Delivery of computing services over the internet with on-demand access and pay-as-you-go pricing.

### CapEx (Capital Expenditure)

Upfront investment in physical infrastructure.

### OpEx (Operational Expenditure)

Pay-as-you-go model used in cloud computing.

### Scalability

Ability to increase or decrease resources.

### Elasticity

Automatic scaling based on demand.

---

## 🌍 Cloud Models

### Public Cloud

Services offered over the internet and shared across customers.

### Private Cloud

Dedicated cloud environment for a single organization.

### Hybrid Cloud

Combination of public and private cloud environments.

---

## 🔐 Shared Responsibility

### Shared Responsibility Model

Defines which security and management tasks are handled by Azure and which by the customer.

---

## 🏗️ Azure Infrastructure

### Region

A geographic area containing multiple datacenters.

### Availability Zone

An isolated datacenter within a region used for high availability.

### Datacenter

A physical facility that houses servers and infrastructure.

---

## ⚙️ Compute

### Virtual Machine (VM)

A virtual server with full control over the operating system.

### Container

A lightweight, portable environment for running applications.

### Azure Functions

Serverless compute service for event-driven execution.

### App Service

Managed platform for hosting web apps and APIs.

---

## 🌐 Networking

### Virtual Network (VNet)

Private network in Azure for resource communication.

### VPN Gateway

Provides secure encrypted connections over the internet.

### ExpressRoute

Private, dedicated connection between on-premises and Azure.

### DNS (Domain Name System)

Resolves domain names to IP addresses.

---

## 🔒 Network Security

### Network Security Group (NSG)

Filters network traffic using allow/deny rules (Layer 4).

### Azure Firewall

Managed, centralized firewall with advanced filtering (Layer 7).

### DDoS Protection

Protects against distributed denial-of-service attacks.

---

## 💾 Storage

### Blob Storage

Stores unstructured data such as files and images.

### File Storage

Provides shared file storage accessible via SMB.

### Queue Storage

Enables messaging between application components.

### Table Storage

NoSQL key-value storage.

---

## 🔁 Redundancy

### LRS (Locally Redundant Storage)

Data replicated within a single datacenter.

### ZRS (Zone-Redundant Storage)

Data replicated across availability zones.

### GRS (Geo-Redundant Storage)

Data replicated to a secondary region.

### GZRS (Geo-Zone-Redundant Storage)

Combines zone and geo redundancy.

---

## 📊 Management & Governance

### Azure Monitor

Collects and analyzes metrics and logs.

### Log Analytics

Tool for querying and analyzing log data.

### Azure Advisor

Provides recommendations for cost, performance, and security.

### Service Health

Shows Azure service issues affecting your resources.

### Resource Health

Provides status of individual resources.

---

## 💰 Cost Management

### Pricing Calculator

Estimates Azure costs before deployment.

### Azure Cost Management

Analyzes and monitors actual usage and costs.

### Tags

Key-value pairs used to organize and track resources.

---

## 🛡️ Governance & Compliance

### Azure Policy

Enforces organizational rules and standards.

### Resource Locks

Prevent accidental deletion or modification.

### Microsoft Purview

Data governance and compliance solution.

---

## 🔐 Identity & Security

### Microsoft Entra ID (Azure AD)

Cloud-based identity and access management service.

### Authentication

Verifying identity (e.g., login).

### Authorization

Granting access to resources.

### RBAC (Role-Based Access Control)

Controls access based on roles.

### Conditional Access

Applies access policies based on conditions (e.g., location, device).

### Zero Trust Model

Security approach: never trust, always verify.

---

## 🛠️ Tools & Deployment

### Azure Portal

Web-based interface for managing Azure resources.

### Azure CLI

Command-line tool for managing Azure.

### PowerShell

Scripting environment for automation.

### ARM Templates

JSON-based templates for infrastructure as code.

### Azure Arc

Extends Azure management to on-premises and other clouds.

---

## ☁️ Service Models

### IaaS (Infrastructure as a Service)

Provides virtualized infrastructure (VMs, storage, networking).

### PaaS (Platform as a Service)

Provides platform for application development without managing infrastructure.

### SaaS (Software as a Service)

Provides fully managed software over the internet.

---

## 🎯 Tip

Focus on understanding:

* Differences between similar terms
* When to use each service
* Scope (global vs resource vs network)
