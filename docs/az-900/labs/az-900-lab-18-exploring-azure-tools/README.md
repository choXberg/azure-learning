# AZ-900 Lab 18 – Exploring Azure Tools (Compliance)

## 📘 Lab Overview
In this lab, you explore Microsoft’s **compliance and trust tools**, including:

- Compliance Offerings documentation  
- Service Trust Portal (STP)  
- Compliance Manager  

These tools help organizations understand how Microsoft ensures **security, privacy, and regulatory compliance** in Azure.

---

## 🎯 Objectives
- Explore **Microsoft compliance offerings**
- Understand the purpose of the **Service Trust Portal**
- Learn how **Compliance Manager** supports risk assessment
- Get familiar with Azure’s compliance transparency

---

## 🧠 Key Concepts

### 🛡️ Compliance in Azure
Microsoft provides:
- Certifications (e.g., ISO, GDPR)
- Audit reports
- Transparency on security practices

---

### 📚 Compliance Categories
Compliance offerings are grouped into:

- Global (e.g., ISO 27001)
- US Government
- Industry (e.g., healthcare, finance)
- Regional (e.g., EU-specific regulations)

---

### 🔍 Service Trust Portal (STP)
The STP provides:
- Audit reports
- Compliance documentation
- Security and privacy details

---

### 📊 Compliance Manager
A tool to:
- Assess compliance posture
- Manage risks
- Track regulatory requirements
- Generate reports

---

## 🧪 Lab Steps Summary

### 1. Access Compliance Offerings
- Open:
  https://docs.microsoft.com/en-us/compliance/regulatory/offering-home

Explore:
- Categories (Global, Industry, Regional)
- Example: **ISO 27001**

💡 Observation:
- Azure provides detailed compliance documentation for each standard

---

### 2. Access Service Trust Portal (STP)
- Open:
  https://servicetrust.microsoft.com/

Explore:
- Audit Reports section
- Certifications and attestations

⚠️ Note:
- Login required for detailed reports

---

### 3. Explore Compliance Manager
- Available within STP
- Navigate to:
  - Audit Reports
  - Compliance tools

💡 Features:
- Risk assessment dashboards
- Compliance scoring
- Actionable recommendations

---

## 🔍 Observations

- Microsoft provides **extensive compliance transparency**
- Audit reports are **independently verified**
- Tools support **enterprise compliance requirements**
- Access to detailed reports requires authentication

---

## ⚠️ Important Notes

- Compliance is a **shared responsibility**
- Microsoft ensures infrastructure compliance
- Customers are responsible for:
  - Data
  - Access control
  - Configuration

---

## 🧾 Use Cases

- Regulatory compliance (e.g., GDPR, ISO)
- Risk assessment for cloud adoption
- Security audits
- Enterprise governance
