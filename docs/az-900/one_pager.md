# AZ-900 One Pager – Final Review

## ☁️ Cloud Basics

* Cloud = on-demand resources over internet
* OpEx (pay-as-you-go), no upfront cost
* Scalable & elastic

---

## 🌍 Cloud Models

* Public → shared, most common
* Private → dedicated
* Hybrid → mix

---

## 🔐 Shared Responsibility

* Azure → infrastructure
* You → data, access, configuration
* More control = more responsibility

---

## 🏗️ Infrastructure

* Region = multiple datacenters
* Availability Zone = isolated datacenter
* Used for high availability

---

## ⚙️ Compute

* VM → full control (IaaS)
* Containers → lightweight apps
* Functions → serverless
* App Service → web apps/API

---

## 🌐 Networking

* VNet → private network
* VPN → secure internet connection
* ExpressRoute → private connection
* DNS → name resolution
* NSG → traffic filtering

---

## 💾 Storage

* Blob → files/images
* File → shared drive
* Queue → messaging
* Table → NoSQL

### Redundancy

* LRS → single DC
* ZRS → zones
* GRS → geo
* GZRS → zones + geo

---

## 📊 Cost

* Pricing Calculator → estimate
* Cost Management → analyze
* Tags → organize

---

## 🛡️ Governance

* Policy → enforce rules
* Locks → prevent deletion
* Purview → data governance

---

## 📈 Monitoring

* Monitor → metrics/logs
* Advisor → recommendations
* Service Health → Azure issues
* Resource Health → resource status

---

## 🛠️ Tools

* Portal → GUI
* CLI / PowerShell → automation
* ARM → IaC
* Arc → hybrid

---

## ⚠️ Exam Must-Know

* Zone ≠ Region
* NSG ≠ Firewall
* Service Health ≠ Resource Health
* IaaS ≠ PaaS ≠ SaaS

---

## 🎯 Strategy

* Think in **use cases**
* Not implementation details
